# Project Constitution

Agents and humans should follow these rules when planning or implementing.

1. Source of truth for intent is the `specs/` directory.
2. Do not invent user-facing scope that is not in the specs.
3. Prefer small, reversible implementation steps.
4. Maintain alignment between code, tests, and acceptance criteria.
5. Mark unresolved ambiguity explicitly instead of guessing silently.
6. Keep changes traceable to a task whenever possible.
7. Update docs when behavior changes.
